Use this API endpoint to cancel a specific booking identified by its id. This is useful when a user needs to free up their schedule or remove an event. For instance, call this API if a user decides to cancel a meeting or appointment due to a scheduling conflict. The only required parameter is the id to accurately identify and cancel the intended event. This action ensures that the slot becomes available again for new bookings or adjustments, helping maintain an up-to-date schedule for the user.
